# 云服务器同步指南

本文说明 **当前推荐** 的发布与同步方式。前端走 GitHub Pages，后端 API 跑在云服务器，二者分开更新。

## 架构概览

| 部分 | 部署位置 | 更新方式 |
|------|----------|----------|
| 前端（React） | GitHub Pages | `git push origin main`，Actions 自动构建部署 |
| 后端 API（Gin） | 云服务器 `124.221.5.162` | SFTP 上传二进制 + 配置文件，再重启服务 |
| 素材清单 / 映射 | 云服务器 + 本地仓库 | SFTP 覆盖 JSON，或上传工具勾选自动同步 |

- 公网 API：`https://api.jadot.cn`
- 远程项目根目录：`/opt/jiadian-hub/app`
- systemd 服务：`jiadian-api.service`
- 后端监听端口：`8080`（见 `backend/.env`）

---

## 重要原则

### 可以做

- 本地改代码 → `git commit` → `git push origin main`（触发前端自动部署）
- 用 **SFTP** 直接覆盖云服务器上的配置文件
- 后端有改动时，重新编译 `jiadian-api` 后 **SFTP 上传二进制**
- 上传完成后执行 `systemctl restart jiadian-api.service`

### 不要做

- **不要在云服务器上对生产目录执行 `git stash -u && git pull`**

  `backend/jiadian-api` 是可执行文件，未纳入 git。使用 `git stash -u` 会把它一起收进 stash，导致服务启动失败（`203/EXEC`），nginx 返回 **502**，前端提示接口不可达。

- 不要指望 `git pull` 更新后端二进制（服务器未安装 Go，且二进制不在仓库中）

## 场景一：只更新素材（resources / 映射表）

适用于：上传新 GIF、视频、图片后，刷新线上资源列表。

### 方式 A：素材上传 GUI（推荐）

1. 运行 `python tools/image_uploader_gui.py`
2. 上传素材
3. 勾选 **「上传后自动同步到云服务器」**
4. 填写云同步配置（主机 / 用户 / 密码 / 远程路径）
5. 如需立即生效，勾选 **「同步后执行重启命令」**

GUI 会自动 SFTP 上传以下文件：

```
src/data/resources.json          → /opt/jiadian-hub/app/src/data/resources.json
backend/config/image_map.json    → /opt/jiadian-hub/app/backend/config/image_map.json
backend/config/resource_map.json → /opt/jiadian-hub/app/backend/config/resource_map.json
```

### 方式 B：命令行脚本

```bash
# 先设置环境变量（不要提交到 git）
set REMOTE_SYNC_HOST=124.221.5.162
set REMOTE_SYNC_USER=ubuntu
set REMOTE_SYNC_PASSWORD=你的密码
set REMOTE_SYNC_BASE_PATH=/opt/jiadian-hub/app

python tools/sync_cloud.py
```

只同步配置、并重启 API：

```bash
python tools/sync_cloud.py --restart
```

## 场景二：只更新前端代码

适用于：改 UI、预览样式、WebUSB 逻辑等 **不涉及后端** 的变更。

```bash
git add .
git commit -m "你的说明"
git push origin main
```

推送后 GitHub Actions（`.github/workflows/deploy-pages.yml`）会自动构建并部署到 GitHub Pages，约 1–2 分钟生效。

**无需**登录云服务器。

---

## 场景三：更新后端 Go 代码

适用于：改了 `backend/main.go`、`backend/service/*` 等 API 逻辑。

### 1. 本地交叉编译 Linux 二进制

在项目根目录执行（Windows PowerShell 示例）：

```powershell
$env:GOOS = "linux"
$env:GOARCH = "amd64"
$env:CGO_ENABLED = "0"
go build -o backend/jiadian-api ./backend
```

确认是 Linux 可执行文件（应以 `ELF` 开头，不是 Windows PE）。

### 2. SFTP 上传二进制并重启

```bash
python tools/sync_cloud.py --binary --restart
```

或手动上传：

```
本地  backend/jiadian-api
  →  远程  /opt/jiadian-hub/app/backend/jiadian-api
```

上传后远程执行：

```bash
chmod +x /opt/jiadian-hub/app/backend/jiadian-api
sudo systemctl restart jiadian-api.service
```

### 3. 提交源码到 GitHub（可选但建议）

```bash
git add backend/
git commit -m "fix: ..."
git push origin main
```

注意：`backend/jiadian-api` 已在 `.gitignore` 中，**不要**把二进制 commit 进仓库。

## 场景四：完整发布（前端 + 后端 + 素材）

按顺序执行：

1. **本地开发与测试**
2. **提交并推送前端/源码**
   ```bash
   git push origin main
   ```
3. **同步云服务器配置**（若素材有变）
   ```bash
   python tools/sync_cloud.py --restart
   ```
4. **上传后端二进制**（若 Go 代码有变）
   ```bash
   python tools/sync_cloud.py --binary --restart
   ```
5. **验证**（见下文）

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `REMOTE_SYNC_HOST` | `124.221.5.162` | 云服务器 IP |
| `REMOTE_SYNC_USER` | `ubuntu` | SSH 用户名 |
| `REMOTE_SYNC_PASSWORD` | （必填） | SSH 密码 |
| `REMOTE_SYNC_BASE_PATH` | `/opt/jiadian-hub/app` | 远程项目根目录 |
| `REMOTE_RESTART_CMD` | `systemctl restart jiadian-api.service` | 重启命令（脚本会自动加 sudo） |

可在系统环境变量或本地 `.env`（勿提交）中配置。素材上传 GUI 也读取同名变量。

## 验证是否成功

在云服务器或本地执行：

```bash
# 服务状态应为 active
systemctl is-active jiadian-api.service

# 本机 API 应返回 200
curl -sS -o /dev/null -w "%{http_code}" http://127.0.0.1:8080/api/verify-token

# 公网 API 应返回 200
curl -sS -o /dev/null -w "%{http_code}" https://api.jadot.cn/api/verify-token
```

前端页面刷新后，资源列表与下载/鉴权应恢复正常。

## 常见问题

### 接口不可达 / 502 Bad Gateway

1. 检查服务：`systemctl status jiadian-api.service`
2. 若日志为 `status=203/EXEC`，说明 **`backend/jiadian-api` 缺失或不可执行**
3. 重新 SFTP 上传二进制并 `chmod +x`，再重启服务

### git pull 后 API 挂了

原因是 stash 带走了未跟踪的二进制。修复：

```bash
python tools/sync_cloud.py --binary --restart
```

### 素材已上传 COS 但线上看不到

1. 确认 `resources.json` 与两个 map 已 SFTP 到云服务器
2. 重启 `jiadian-api.service`
3. 前端强制刷新（GitHub Pages 可能有缓存）

## 文件对照表

| 本地路径 | 远程路径 | 何时需要同步 |
|----------|----------|--------------|
| `src/data/resources.json` | `.../src/data/resources.json` | 增删改素材 |
| `backend/config/image_map.json` | `.../backend/config/image_map.json` | 图片/封面映射变更 |
| `backend/config/resource_map.json` | `.../backend/config/resource_map.json` | 视频/GIF/软件映射变更 |
| `backend/jiadian-api` | `.../backend/jiadian-api` | Go 后端代码变更 |
| `backend/.env` | `.../backend/.env` | 极少改动，**仅在服务器上维护，勿覆盖** |

前端构建产物（`dist/`）不部署到云服务器，由 GitHub Pages 托管。
